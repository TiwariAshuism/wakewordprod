# AURA v1 — Device Runbook (Steps 0, 2, 3)

These steps require an Android toolchain **and a physical device** that the build/CI
environment used for v1 did not have (no SDK/NDK/CMake/Gradle, JDK 24 not 17, no
device attachable). Run them on a provisioned Android dev machine. Everything else
(real model, front-end alignment, ADRs, host tests) is already done — see
`REPORT_v1.md`.

## Prerequisites (match `tools/versions.txt` exactly — do not build against mismatched versions)

| Tool | Pinned | Check |
|---|---|---|
| JDK | **17** | `java -version` — must be 17 (AGP 8.7.2 rejects JDK 24) |
| Android SDK | compile/target **35** | `sdkmanager --list \| grep platforms` |
| NDK | **27.2.12479018** | `sdkmanager --list \| grep ndk` |
| CMake | **3.22.1** | `sdkmanager --list \| grep cmake` |
| Gradle | 8.11.1 (via committed wrapper) | `./gradlew --version` |

```bash
export ANDROID_HOME=/path/to/Android/Sdk        # and add platform-tools to PATH
export JAVA_HOME=/path/to/jdk-17
./tools/doctor.sh                                # sanity vs tools/versions.txt
sdkmanager "platforms;android-35" "ndk;27.2.12479018" "cmake;3.22.1" "build-tools;35.0.0"
```

The Gradle wrapper jar **is committed** in v1 (`gradle/wrapper/gradle-wrapper.jar`,
sha256 `2db75c40782f5e8ba1fc278a5574bab070adccb2d21ca5a6e5ed840888448046`). If you
distrust the committed jar, regenerate: `gradle wrapper --gradle-version 8.11.1`.

## Step 2 — ORT allocation on the RT Audio thread (measure, then report before fixing)

The open question (`ADR-VAD-RTAlloc`): ORT `Run()` allocates internally on the Silero
VAD path, which runs on the Audio thread; v1 minimizes this (reused I/O `OrtValue`s)
but did not fix it. **Measure on-device first**, then send numbers back for the fix
decision. Two approaches, no source change needed:

**A) ORT built-in profiling (per-op + memory).** Temporarily enable in
`core/vad/SileroVad.cpp` (`impl->opts.EnableProfiling("aura_vad_prof")`) OR
`core/runtime/OnnxRuntimeBackend.cpp`, rebuild, run, then pull and inspect:
```bash
adb shell run-as com.getnyx.aura.app ls files      # find aura_vad_prof_*.json
adb exec-out run-as com.getnyx.aura.app cat files/aura_vad_prof_XXXX.json > vad_prof.json
# inspect: per-op dur + the alloc/free events ORT records
```

**B) Native heap profiling with Perfetto `heapprofd` (counts + sizes, per thread).**
```bash
adb shell setprop persist.traced.enable 1
python $ANDROID_HOME/.../heap_profile -n com.getnyx.aura.app -i 1000   # or a Perfetto config
# speak into the mic ~15 s so VAD Run() fires repeatedly, then open the trace in
# ui.perfetto.dev and read malloc count/bytes attributed to the Audio thread during Run().
```

Report: **allocations per `Run()`** (count + total bytes) attributed to the Audio
thread. Then we choose ADR-VAD-RTAlloc (a) arena allocator, (b) IoBinding, or (c)
hand-rolled Silero — do not implement before reporting.

## Step 3 — On-device true/false-positive (Acceptance #2)

**Use a real device, not the emulator** (emulator mic input is not representative —
Stage 9 §8.1). Grant the mic permission when the app launches.

```bash
adb logcat -c && adb logcat -s AURA:V
```

- **True positive:** say **"marvin"** clearly a few times → expect a toast
  ("marvin detected (conf=…)") and a logcat line `wake word CONFIRMED (stage1)` +
  `marvin detected … cid=<CorrelationId>`. Confirm the same `CorrelationId` appears on
  the preceding `VAD triggered` line (Stage 7 §12 tracing end-to-end).
- **Negative:** say 4–5 other words (e.g. "yes", "no", "hello", "computer") and stay
  silent for a stretch → expect **no** detection. This is a sanity check that the real
  model discriminates, not a FA/hr benchmark (that needs the Stage 7 §14 corpus/harness,
  out of scope).

Report both plainly, including partial/failed results. **Do not tune
`DetectConfig.stage1Threshold` or retrain to force a pass without approval.** If it
over- or under-fires, capture: the confidence values logged, whether VAD is gating
correctly, and the model's host TP/FP numbers (`python tools/verify_kws_host.py`) for
comparison, and report back.
